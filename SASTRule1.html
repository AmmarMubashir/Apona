Info.(1202)
Critical

Do not expose internal data and system information to the outside.

Internal data system information help attackers to determine attack targets. For example, when a JRE version is exposed, an attacker can decide what to attack based on the vulnerabilities that exist in that version.

After development, delete all code that outputs internal data system information to the outside. if you need to leave a note about it, do not print it out, but write it to a log file on the server.
